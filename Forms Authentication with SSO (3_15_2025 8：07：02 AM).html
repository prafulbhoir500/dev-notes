Step 1: Configure Web.config for Forms Authentication

Modify your Web.config to enable Forms Authentication:

xml
<configuration>
  <system.web>
    <authentication mode="Forms">
      <forms loginUrl="~/Account/Login.aspx" timeout="2880" slidingExpiration="true" />
    </authentication>
    <authorization>
      <deny users="?" />
    </authorization>
  </system.web>
</configuration>
  • This will redirect unauthenticated users to Login.aspx.

Step 2: Create Authentication Helper

This helper class will handle user login for both SSO and Application-based authentication.

App_Code/AuthenticationHelper.cs

csharp
using System;
using System.Web;
using System.Web.Security;

public class AuthenticationHelper
{
    public static void SignIn(string username, bool isPersistent)
    {
        FormsAuthentication.SetAuthCookie(username, isPersistent);
    }

    public static void SignOut()
    {
        FormsAuthentication.SignOut();
        HttpContext.Current.Session.Abandon();
    }

    public static bool IsUserAuthenticated()
    {
        return HttpContext.Current.User.Identity.IsAuthenticated;
    }
}
  • This helps manage login, logout, and session handling.

namespace WebFormsSSOApp.Account

            if (!string.IsNullOrEmpty(code))

                string tokenEndpoint = identityServerUrl + "/connect/token";
                string clientId = "webforms-app";
                string clientSecret = "your-client-secret";
                string redirectUri = "https://your-webforms-app.com/Account/SSOCallback.aspx";

                using (HttpClient client = new HttpClient())
                {
                    var postData = new FormUrlEncodedContent(new[]
                    {
                        new KeyValuePair<string, string>("client_id", clientId),
                        new KeyValuePair<string, string>("client_secret", clientSecret),

                        new KeyValuePair<string, string>("code", code),

To ensure Single Logout (SLO) across all WebForms applications when one application logs out, you can implement Session-Based Logout Handling and IdentityServer4 Single Sign-Out.

Approach: Centralized Logout Mechanism

When a user logs out from one application, all other applications should automatically log out. This can be done using:

  1. IdentityServer4 Front-Channel Logout:
    • When a user logs out from one WebForms app, it will notify all other applications to log out.
  2. Session Expiration & Forms Authentication Expiry:
    • Use JavaScript polling or Hidden iframe to check session status from the SSO provider.

Step 1: Modify IdentityServer4 Logout to Notify Applications

If you're using IdentityServer4, modify its logout endpoint to send a logout request to all registered applications.

IdentityServer4 Logout Endpoint

Modify IdentityServer Logout Endpoint to include Front-Channel Logout:

csharp
[HttpGet]
public IActionResult Logout()
{
    var idTokenHint = HttpContext.User.FindFirst("id_token")?.Value;
    var logoutUrl = "https://your-webforms-app.com/Account/SSOLogout.aspx";
    
    return SignOut(new AuthenticationProperties
    {
        RedirectUri = logoutUrl
    }, "Cookies", "oidc");
}
  • This will redirect users to WebForms logout URL after logging out from IdentityServer.

Step 2: Create Global Logout Handler in WebForms

This will handle logout requests from IdentityServer and other applications.

Account/SSOLogout.aspx.cs

csharp
using System;
using System.Web;
using System.Web.Security;

namespace WebFormsSSOApp.Account
{
    public partial class SSOLogout : System.Web.UI.Page
    {
        protected void Page_Load(object sender, EventArgs e)
        {
            FormsAuthentication.SignOut();
            HttpContext.Current.Session.Abandon();
            Response.Cookies["ASP.NET_SessionId"].Expires = DateTime.Now.AddYears(-1);
            Response.Cookies[".ASPXAUTH"].Expires = DateTime.Now.AddYears(-1);

            // Redirect to IdentityServer Logout if needed
            string identityServerLogout = "https://your-identityserver.com/connect/endsession";
            Response.Redirect(identityServerLogout);
        }
    }
}
  • This clears the authentication cookies and redirects to IdentityServer Logout.

Step 3: Automatically Detect Logout Using JavaScript

Add this script to Master Page or Default.aspx to detect logout across applications.

html
<script>
    function checkSession() {
        var iframe = document.createElement("iframe");
        iframe.style.display = "none";
        iframe.src = "https://your-identityserver.com/check-session"; 
        document.body.appendChild(iframe);
        
        iframe.onload = function () {
            setTimeout(function () {
                document.body.removeChild(iframe);
                checkSession(); // Check again
            }, 5000); // Poll every 5 seconds
        };
    }

    checkSession();
</script>
  • This script checks if the session is still valid using an iframe.
  • If SSO session is expired, redirect the user to SSOLogout.aspx.
